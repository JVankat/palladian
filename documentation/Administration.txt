== Increasing the version ==
- Checkout the project from the git repository or fetch most recent changes
- switch to branch origin/stable
	- git checkout origin/stable
- check if current version builds successfully
	- mvn clean install
- merge stable branch with most recent version
	- git merge origin/master
- increase version number in all POM.xmls
	- i.e. 0.1.10-SNAPSHOT --> 0.1.10
- install new stable version to nexus repository
	- mvn clean deploy
- tag the current stable version in git
	- git tag -a 0.1.10
- upload everything (including the new tag) to the repository
	- git push -tag
- change current branch to master
	- git checkout origin/master
- update the version in all POM.xmls.
	- i.e. since we are on our way to 0.1.11 now we call the new version in master 0.1.11-SNAPSHOT
- push changes to the repository
	- git push
